# Исправление ошибки "Invalid data" в терминале

## Проблема
После создания веб-сокета в терминале пользователя возникала ошибка "Invalid data", что указывало на проблемы со связью с контейнером.

## Причины
1. Отсутствие проверки авторизации пользователя в обработчиках WebSocket
2. Недостаточная валидация данных сессии
3. Отсутствие проверки статуса контейнера перед выполнением команд
4. Неинформативные сообщения об ошибках на английском языке
5. Отсутствие инициализации сессии терминала при подключении

## Внесенные изменения

### 1. app.py - Улучшенные обработчики WebSocket

#### Добавлена функция инициализации сессии (`terminal_init`):
- Проверяет наличие session_token
- Проверяет авторизацию пользователя
- Извлекает информацию о сессии, контейнере и курсе из БД
- Проверяет статус контейнера (должен быть 'running')
- Отправляет подтверждение инициализации клиенту

#### Улучшен обработчик `handle_terminal_input`:
- Добавлено логирование всех входящих данных
- Проверка авторизации пользователя перед выполнением любых операций
- Добавлена проверка статуса контейнера перед выполнением команды
- Улучшены сообщения об ошибках на русском языке
- Добавлено детальное логирование ошибок выполнения команд

#### Улучшен обработчик `handle_connect`:
- Использует `flask_request.sid` вместо `request.sid` для корректной работы
- Отправляет событие 'connected' клиенту
- Добавлено логирование подключений

### 2. templates/terminal.html - Инициализация сессии на клиенте

Добавлен вызов `socket.emit('terminal_init', {session_token: sessionToken})` сразу после подключения к WebSocket. Это позволяет:
- Проверить валидность сессии до начала работы
- Получить информативное сообщение об ошибке, если контейнер не запущен
- Убедиться, что все данные корректны перед отправкой команд

## Диагностика

При возникновении ошибки теперь в логах сервера будут отображаться:
- `Invalid data received: session_token=..., command=...` - если данные некорректны
- `Session not found: ...` - если сессия не найдена в БД
- `Executing command in VM ...: ...` - успешное начало выполнения команды
- `Decode error: ...` - ошибка декодирования вывода
- `Failed to execute command: ...` - ошибка выполнения команды в Proxmox

## Рекомендации по устранению проблем

1. **Проверьте подключение к Proxmox:**
   - Убедитесь, что конфигурация Proxmox корректна (host, port, user, password)
   - Проверьте доступность сервера Proxmox из контейнера приложения

2. **Проверьте статус контейнера:**
   - Контейнер должен быть в статусе 'running'
   - Если контейнер остановлен, запустите его через панель управления

3. **Проверьте сессию пользователя:**
   - Убедитесь, что пользователь авторизован
   - Проверьте, что session_token корректен и соответствует активной сессии

4. **Включите логирование:**
   - Запустите приложение с флагогом debug для детального логирования
   - Просматривайте server.log для анализа ошибок
